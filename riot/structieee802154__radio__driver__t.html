Detailed Description

IEEE 802.15.4 radio driver API definition.

This is the set of functions that must be implemented by any driver for a 802.15.4 radio transceiver.

radio_tx_status_t(* ieee802154_radio_driver_t::load_tx)(ieee802154_packet_kind_t kind, ieee802154_node_addr_t dest, bool use_long_addr, bool wants_ack, void *buf, unsigned int len)

Load the tranceiver TX buffer with the given IEEE 802.15.4 packet.

Parameters
[in]kindKind of packet to transmit.
[in]destAddress of the node to which the packet is sent.
[in]use_long_addrtrue to use the 64-bit address mode with dest param; false to use "short" PAN-centric mode.
[in]wants_acktrue to request an acknowledgement from the receiving node for this packet; false otherwise.
[in]bufPointer to the buffer containing the payload of the 802.15.4 packet to transmit. The frame header (i.e.: FCS, sequence number, src and dest PAN and addresses) is inserted using values in accord with kind parameter and transceiver configuration.
[in]lenLength (in bytes) of the outgoing packet payload.
Returns
The outcome of this packet's transmission.
