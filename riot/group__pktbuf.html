Detailed Description

A global network packet buffer.

Macro Definition Documentation

#define PKTBUF_SIZE   (6144)

Maximum size of the packet buffer.

The rational here is to have at least space for 4 full-MTU IPv6 packages (2 incoming, 2 outgoing; 2 * 2 * 1280 B = 5 KiB) + Meta-Data (roughly estimated to 1 KiB; might be smaller)

int pktbuf_copy ( void *  pkt,
const void *  data,
size_t  data_len 
)

Copies packet data into the packet buffer, safely.

Use this instead of memcpy, since it is thread-safe and checks if pkt is

  1. in the buffer at all
  2. its size is smaller or equal to the data allocated at pkt

If the pkt is not in the buffer the data is just copied as memcpy would do.

Parameters
[in,out]pktThe packet you want to set the data for.
[in]dataThe data you want to copy into the packet.
[in]data_lenThe length of the data you want to copy.
Returns
data_len, on success.
-EFAULT, if data is NULL and DEVELHELP is defined.
-EINVAL, if pkt is NULL and DEVELHELP is defined.
-ENOBUFS, if data_len was greater than the packet size of pkt.
void pktbuf_hold ( const void *  pkt)

Marks the data as being processed.

Internally this increments just a counter on the data. pktbuf_release() decrements it. If the counter is <=0 the reserved data block in the buffer will be made available again.

Parameters
[in]pktThe packet you want mark as being processed.

void* pktbuf_realloc ( const void *  pkt,
size_t  size 
)

Reallocates new space in the packet buffer, without changing the content.

If enough memory is available behind it or size is smaller than the original size the packet will not be moved. Otherwise, it will be moved. If no space is available nothing happens.

Parameters
[in]pktOld position of the packet in the packet buffer
[in]sizeNew amount of data you want to allocate
Returns
Pointer to the (maybe new) position of the packet in the packet buffer, on success.
NULL, if no space is left in the packet buffer or size was 0. The packet will remain at ptr.
void pktbuf_release ( const void *  pkt)

Marks the data as not being processed.

Parameters
[in]pktThe packet you want mark as not being processed anymore.

Internally this decrements just a counter on the data. pktbuf_hold() increments and any allocation operation initializes it. If the counter is <=0 the reserved data block in the buffer will be made available again.
