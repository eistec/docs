Detailed Description

Basic general interface to communicate with a network protocol.

The idea of this module is that every network layer provides a basic set of commands to communicate with its lower and upper layer and the system. In this model every layer has some sort of control thread with which the other layers and the system communicates (illustrated as cohesive boxes below). 

   +-----+  +-----+   +-----+
   | App |  | App |   | App |
   +-----+  +-----+   +-----+
      ^        ^         ^
    netapi   netapi    netapi   // alternatively the normal socket API
      v        v         v
  +--------------------------+
  | Socket | Socket | Socket |
  |      Transport layer     |
  +--------------------------+
               ^
             netapi
               v
  +--------------------------+
  |      Network layer       |
  +--------------------------+
     ^         ^         ^
   netapi    netapi    netapi
     v         v         v
 +-------+ +-------+ +-------+
 |  MAC  | |  MAC  | |  MAC  |
 +-------+ +-------+ +-------+
 | Radio | | Radio | | Radio |
 +-------+ +-------+ +-------+

Enumeration Type Documentation

enum netapi_cmd_type_t

Basic communication types with a network layer.

This list may be expanded by a some layer (using values > SND_PKT + 4), but no guarantee can be given, that other layers interpret it exactly as the former. The values are chosen to be as compatible as possible with the legacy tranceiver module.

Enumerator
NETAPI_CMD_RCV 

Packet was received from lower layer.

NETAPI_CMD_SND 

Packet was send from upper layer or is send from this layer to lower layer.

NETAPI_CMD_GET 

Get an option of this layer.

NETAPI_CMD_SET 

Set an option of this layer.

NETAPI_CMD_REG 

Registers an upper layer control thread to this layer for receiving.

If there is no space left in the control threads registry, the result of the acknowledgement must be errno -ENOBUFS.

NETAPI_CMD_UNREG 

Unregisteres a previously receiving upper layer control thread to this layer.

Control thread does not need to notify, if the thread in question was previously known. Unknown PIDs are just ignored.

NETAPI_CMD_ACK 

Acknowledges to a previous command.

enum netapi_conf_type_t

Basic configuration types for a network layer.

Developers of new network protocols are adviced to choose new more specific names for this values in the specific layer (e. g. NETAPI_CONF_ADDRESS may be called something port-related in the Transport layer). Layer specific options might also be defined. The values are chosen to be as compatible as possible with the legacy tranceiver module. Every protocol must supply the format of the types for this options but interpret them always optimisticly.

A protocol should at least always answer with an acknowledgement of result Basic network interface NETAPI_STATUS_OK if the operation was successfull or with errno -ENOTSUP if the operation is not supported.

Enumerator
NETAPI_CONF_PROTO 

Set or get the protocol for the layer as of type netdev_proto_t.

NETAPI_CONF_CARRIER 

Set or get a carrier for the layer (e.

g. channel in radios or L3 protocol for transport layer) as unsigned value.

NETAPI_CONF_ADDRESS 

Set or get a address for the layer (e.

g. address in network or link layer, port in transport layer, ...) as unsigned value or array of unsigned values.

NETAPI_CONF_SUBNETS 

Set or get a sub-nets for the layer (e.

g. sub-net in IPv4, prefixes in IPv6, PAN id in IEEE 802.15.4).

NETAPI_CONF_MAX_PACKET_SIZE 

Set or get the maximum packet size for the layer.

NETAPI_CONF_SRC_LEN 

Set or get default address length.

NETAPI_CONF_REGISTRY 

Get (for setting always use NETAPI_CMD_REG and NETAPI_CMD_UNREG) all PIDs of currently registered threads as array of kernel_pid_t.
