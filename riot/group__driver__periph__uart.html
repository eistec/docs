Initialize a given UART device.

The UART device will be initialized with the following configuration:

  • 8 data bits
  • no parity
  • 1 stop bit
  • baud-rate as given
Parameters
[in]uartthe UART device to initialize
[in]baudratethe desired baud-rate in baud/s
[in]rx_cbreceive callback is called for every byte the is receive in interrupt context
[in]tx_cbtransmit callback is called when done with sending a byte (TX buffer gets empty)
[in]argoptional argument passed to the callback functions
Returns
0 on success
-1 for invalid baud-rate
-2 for all other errors
int uart_init_blocking ( uart_t  uart,
uint32_t  baudrate

Initialize an UART device for (conventional) blocking usage.

This function initializes the an UART device for usage without interrupts. When initializing with this function, the corresponding read_blocking and write_blocking functions must be used.

The blocking mode should only be used for debugging and testing.

Same as uart_init(), the UART device is configured with in 8N1 mode with the given baud-rate.

Parameters
[in]uartthe UART device to initialize
[in]baudratethe desired baud-rate in baud/s
Returns
0 on success, -1 for invalid baud-rate, -2 for all other errors

Write a byte into the UART's send register.

Writing a byte into while another byte is still waiting to be transferred will override the old byte. This method should be used in the transmit callback routine as in this it is made sure that no old byte is waiting to be transferred.
