)

Adds a prefix information to an interface.

If it already exists, the values valid_lifetime, preferred_lifetime, advertisable, and flags will be updated accordingly and the prefix will be marked as in_use.

See also
RFC 4861, section 4.6.2 .
Parameters
[in]if_idThe interface's ID.
[in]prefixThe prefix.
[in]prefix_lenThe length of the prefix in bit.
[in]valid_lifetimeThe time in seconds this prefix is valid for on-link determination. NDP_OPT_PI_VLIFETIME_INFINITE for infinite lifetime.
[in]preferred_lifetimeThe time in seconds addresses generated with this prefix remain preferred. NDP_OPT_PI_PLIFETIME_INFINITE for infinite lifetime.
[in]advertisableSet this to a value != 0 to advertise this prefix information with the Prefix Information Option, set it to 0 if not.
[in]flagsFlags for the Prefix Information Option. Valid values are ICMPV6_NDP_OPT_PI_FLAG_ON_LINK and ICMPV6_NDP_OPT_PI_FLAG_AUTONOM
