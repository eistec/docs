Detailed Description

Network device drivers.

The module the network device drivers. It consists of a set of device independent interface definitions and specific implementations for every device.

enum netdev_opt_t

Definition of basic network device options.

Note
Feel free to expand if your device needs/supports more.
Enumerator
NETDEV_OPT_PROTO 

Communication type for the device as defined by netdev_proto_t.

If a driver does not support the type (but the setting of the option is supported) it netdev_driver_t::set_option() shall result with -EPROTONOSUPPORT.

value_len for netdev_driver_t::get_option() must always be at least sizeof(netdev_proto_t).

NETDEV_OPT_CHANNEL 

Channel for the device as unsigned value in host byte order.

NETDEV_OPT_ADDRESS 

Hardware address for the device as unsigned value in host byte order.

NETDEV_OPT_NID 

Network ID (e.g.

PAN ID in IEEE 802.15.4) for the device as unsigned value in host byte order

NETDEV_OPT_ADDRESS_LONG 

Longer hardware address for the device (e.g.

EUI-64) for the device as unsigned value in host byte order

NETDEV_OPT_TX_POWER 

The output of the device in dB as signed value in host byte order.

NETDEV_OPT_MAX_PACKET_SIZE 

Maximum packet size the device supports unsigned value in host byte order.

NETDEV_OPT_SRC_LEN 

Default mode the source address is set to as value of size_t.

(e.g. either PAN-centric 16-bit address or EUI-64 in IEEE 802.15.4)

NETDEV_OPT_LAST 

Last value for netdev_opt_t defined here.

Specific devices or modules like Basic network interface that utilize these values to may define higher values, but they must be greater or equal to NETDEV_OPT_LAST.

Transmit the given IEEE 802.15.4 packet, by calling functions netdev_802154_driver_t::load_tx() and netdev_802154_driver_t::transmit() successfully.

Default value for netdev_802154_driver_t::send()

Parameters
[in]devthe network device
[in]kindKind of packet to transmit.
[in]destAddress of the node to which the packet is sent.
[in]use_long_addr1 to use the 64-bit address mode with dest param; 0 to use "short" PAN-centric mode.
[in]wants_ack1 to request an acknowledgement from the receiving node for this packet; 0 otherwise.
[in]upper_layer_hdrsheader data from higher network layers from highest to lowest layer. Must be prepended to the data stream by the network device. May be NULL if there are none.
[in]bufPointer to the buffer containing the payload of the 802.15.4 packet to transmit. The frame header (i.e.: FCS, sequence number, src and dest PAN and addresses) is inserted using values in accord with kind parameter and transceiver configuration.
[in]lenLength (in bytes) of the outgoing packet payload.

wraps netdev_802154_driver_t::send(), default value for netdev_802154_driver_t::send_data().

Parameters
[in]devthe network device
[in]destthe (hardware) destination address for the data in host byte order.
[in]dest_lenthe length of dest in byte
[in]upper_layer_hdrsheader data from higher network layers from highest to lowest layer. Must be prepended to the data stream by the network device. May be NULL if there are none.
[in]datathe data to send
[in]data_lenthe length of data in byte
Returns
the number of byte actually (data_len + total length of upper layer headers) send on success
-EAFNOSUPPORT if address of length dest_len is not supported by the device dev
-EBUSY if transmission cannot start because radio medium is already busy or collision on radio medium occured.
-EINVAL if wrong parameter was given
-ENODEV if dev is not recognized as IEEE 802.15.4 device
-EMSGSIZE if the total frame size is too long to fit in a frame of the device dev
-EIO if any other occured (netdev_802154_driver_t::send() returned NETDEV_802154_TX_STATUS_ERROR)
